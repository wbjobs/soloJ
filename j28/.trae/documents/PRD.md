# 混沌工程实验平台 PRD

## 1. 项目概述

### 1.1 项目背景
混沌工程是一种通过在分布式系统上进行实验的方法，以建立系统在生产环境中承受动荡条件的信心。本平台旨在提供一个可视化、可管控的混沌实验管理系统。

### 1.2 项目目标
- 通过YAML配置文件定义多样化的实验场景
- 支持Consul/Etcd服务发现，自动发现目标服务
- 记录实验前后的业务指标和系统指标
- 提供实验回滚机制，确保故障可自动停止
- Web界面展示实验列表、实时指标对比图表
- 支持实验计划的审批流程

### 1.3 目标用户
- SRE工程师：执行混沌实验，验证系统稳定性
- 开发人员：测试服务容错能力
- 管理员：审批实验计划，管理平台权限

## 2. 功能需求

### 2.1 实验场景配置
- YAML格式配置文件定义实验
- 支持延迟注入（如对订单服务注入5秒100%延迟）
- 支持Pod杀掉（如随机杀掉支付服务的1个Pod）
- 支持CPU/内存资源占用
- 支持异常注入（如返回指定错误码）

### 2.2 服务发现集成
- Consul集成：从Consul获取服务实例列表
- Etcd集成：从Etcd获取服务注册信息
- 手动配置：支持手动输入目标服务地址

### 2.3 实验执行与监控
- 自动生成唯一实验ID
- 记录注入前基线指标（RPS、P99延迟、错误率）
- 实时监控实验过程中的指标变化
- 记录系统指标（CPU使用率、内存使用率）
- 实验结束后生成对比报告

### 2.4 实验回滚机制
- 一键停止实验
- 自动回滚阈值（如错误率超过50%自动停止）
- 实验超时自动终止
- 恢复目标服务正常状态

### 2.5 实验审批流程
- 用户提交实验申请
- 管理员审批/拒绝
- 审批通过后才能执行
- 审批历史记录

### 2.6 Web界面功能
- 实验列表展示（状态、时间、创建人）
- 实验详情页面（配置、指标、日志）
- 实时指标对比图表（ECharts）
- 实验创建向导
- 审批管理面板

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 < 2秒
- 指标数据实时刷新（5秒间隔）
- 支持同时运行10个并发实验

### 3.2 安全要求
- 用户身份认证
- 基于角色的权限控制
- 实验操作审计日志

### 3.3 可用性要求
- 平台自身99.9%可用性
- 实验失败不影响平台运行

## 4. 技术栈

### 4.1 后端
- Java 17
- Spring Boot 3.x
- Chaos Monkey库
- MySQL 8.0
- Consul/Etcd客户端

### 4.2 前端
- Vue 3
- ECharts
- Element Plus
- Axios

## 5. 用户界面设计

### 5.1 整体风格
- 深色主题，科技感设计
- 清晰的数据可视化
- 专业的运维平台风格

### 5.2 主要页面
1. 仪表盘：总览实验状态、关键指标
2. 实验列表：所有实验的表格展示
3. 实验详情：配置信息、实时图表、执行日志
4. 创建实验：YAML编辑器 + 表单配置
5. 审批中心：待审批实验列表
6. 服务管理：已发现的服务列表
